02 · Fondos oscuros — Comparación

¿Qué tan oscuro es "oscuro"?

VAR(--DARK)

Identificá dónde está el mayor impacto.

Este es el fondo principal para secciones oscuras. Profundo, casi negro, con apenas un tinte violeta. Transmite seriedad enterprise.

#1A0A2E ✓ CORRECTO — CTA, footer, hero oscuro
VARIANTE DENSA

La IA ya llegó a tu empresa. Ahora hay que hacerla funcionar.

Variante un poco más violácea, pero sigue siendo muy oscura. Usar en slides y secciones pre-AI.

#1A0F30 ✓ ACEPTABLE — slides, variación sutil
VAR(--ACCENT-DARK)

Este fondo es demasiado violeta y claro.

Este color NO es para fondos de página completa. Es para headlines, botones y elementos focales. Usado como fondo se ve infantil, no profesional.

#2D1B4E ✗ PROBLEMA — No usar como fondo de sección
VAR(--ACCENT-DEEP)

Esto es directamente inaceptable.

Un gris violeta medio. Parece una app de meditación de 2018. Cero autoridad enterprise.

#5E5070 ✗ PROHIBIDO — Nunca como fondo
🚨

Regla clara: Para fondos oscuros de página/sección, SOLO usar #1A0A2E (--dark) o #1A0F30 (variante densa).
Los colores #2D1B4E y #5E5070 son para elementos UI (headlines, botones, cards invertidas), NO para backgrounds completos.

03 · Amarillo — Reglas de uso

El amarillo tiene reglas estrictas.

✓ CORRECTO

Texto con acento amarillo

El amarillo #F4EB33 sobre fondo #1A0A2E. Alto contraste, legible, premium.

✗ INCORRECTO

Texto con acento amarillo

El amarillo sobre fondo claro es ilegible. No tiene suficiente contraste. Nunca usarlo así.

Texto ilegible ↗
✓ CORRECTO
Próximo paso

Eyebrow label con guiones amarillos sobre fondo oscuro denso. Perfecto.

✗ INCORRECTO
Invisible label

El amarillo como label sobre blanco desaparece completamente.

08 · Secciones completas

Patrones de sección armados.

El Desafío

El 87% de los proyectos de IA
nunca llegan a producción.

No porque la tecnología falle, sino porque nadie se ocupó de conectarla con los procesos reales del equipo.

01

Sin estrategia clara

Se compran herramientas sin entender qué problema resuelven ni cómo se integran a la operación existente.

Inversión sin retorno.
02

Datos fragmentados

La información vive en silos. Nadie tiene una vista unificada para tomar decisiones informadas.

Decisiones a ciegas.
03

Equipos desconectados

La IA se implementa como un proyecto aislado de IT, sin ownership del negocio ni métricas de éxito claras.

La IA muere en un POC.
